# 📎 全视维 Web Clipper 使用手册（小白版）

> 这是一份"看完就会用"的图文步骤说明。哪怕你完全没用过浏览器扩展、没听说过剪藏，也照着走就行。
>
> **它能做什么？** 在浏览器看到好文章/资料，点一下就保存到电脑上的「全视维」桌面端，自动整理到今日 journal 或新页面。不依赖任何云服务、不需要登录账号、数据全在你自己电脑上。

---

## 一、准备工作（30 秒确认 3 件事）

在开始之前，请确认：

| 项目 | 怎么看 | 必须 |
|------|--------|------|
| ① 桌面端已安装并能正常打开 | 双击「全视维」图标能看到主界面 | ✅ |
| ② 已经打开过一个图谱 | 主界面能看到笔记列表，不是空白欢迎页 | ✅ |
| ③ 浏览器是 Chrome / Edge / 国产 Chromium 系（夸克、QQ、360 极速、火狐等） | 在浏览器右上角能看到拼图状的「扩展」图标 | ✅ |

> ⚠️ Safari 用户：本扩展暂不支持 Safari，请用 Chrome 或 Edge。

---

## 二、下载并安装浏览器扩展（5 步搞定）

### 第 1 步：下载 zip 安装包

打开下面这个链接，浏览器会直接开始下载 zip：

👉 https://github.com/jabc9543-byte/-/releases/download/v0.1.9/quanshiwei-web-clipper-v0.1.9.zip

也可以打开 Release 页面手动找到 `quanshiwei-web-clipper-v0.1.9.zip` 点下载：
https://github.com/jabc9543-byte/-/releases/tag/v0.1.9

### 第 2 步：解压到一个**别动它**的位置

- Windows：右键 zip → 「全部解压缩」→ 选一个**永久存放**的位置（比如 `D:\我的工具\全视维-Clipper\`）。
- 注意：**解压后的这个文件夹以后不能删，也不能移走**！删了浏览器扩展就坏了。

解压后文件夹里应该有这些东西：

```
全视维-Clipper\
├── manifest.json
├── background.js
├── content.js
├── popup.html
├── popup.css
├── popup.js
├── README.md
└── icons\
    ├── icon-16.png
    ├── icon-48.png
    └── icon-128.png
```

### 第 3 步：打开浏览器的"扩展管理"页

在浏览器**地址栏**输入下面对应的地址，回车：

- Chrome：`chrome://extensions`
- Edge：`edge://extensions`
- QQ / 360 极速 / 夸克等：通常也是 `chrome://extensions`
- 火狐（Firefox）：`about:debugging#/runtime/this-firefox`，然后看下文「火狐用户特别说明」

### 第 4 步：打开"开发者模式"

进入扩展页面后：

- Chrome / Edge：在页面**右上角**找到「开发者模式」开关，**点开它**（按钮变蓝/变绿）。
- 开关打开后，页面左上角会多出三个按钮：「加载已解压的扩展程序」「打包扩展程序」「更新」。

### 第 5 步：加载扩展

1. 点击「**加载已解压的扩展程序**」（英文是 *Load unpacked*）。
2. 在弹出的文件夹选择窗口中，**选中第 2 步解压出来的那个文件夹**（包含 manifest.json 的那一层），点「选择文件夹」。
3. 加载成功后，扩展列表里会出现「**全视维 Web Clipper**」，并且默认开启。
4. 浏览器右上角工具栏会多一个**蓝色方块**图标，就是这个扩展。

> 找不到图标？点工具栏的**🧩 拼图图标** → 在列表里找到「全视维 Web Clipper」→ 点旁边的**📌 图钉**把它固定到工具栏。

### 火狐用户特别说明

火狐加载方式略不同：
1. 地址栏输入 `about:debugging#/runtime/this-firefox`
2. 点击「临时载入附加组件...」
3. 选择解压目录里的 `manifest.json` 文件
4. ⚠️ 火狐的临时加载在浏览器**重启后会失效**，每次重启都要重新加载一次。

## 三、连接到桌面端（最关键的 2 个数字一样要对得上）

### 第 1 步：在桌面端取 token

1. 打开「全视维」桌面端。
2. 左侧（或顶部菜单）找到「**插件**」入口 → 点开「**Clipper**」面板。
   - 如果看不到「Clipper」，先去「插件市场」找到 **Web Clipper Pro** 点击「安装」（绿色按钮）。
3. 在 Clipper 面板顶部能看到「**X-Clip-Token**」一栏，点旁边的「**复制**」按钮。
   - 如果显示是 `••••••••`，先点「**显示**」让它变成真实字符串，再「复制」。
   - 如果觉得 token 泄露了，点「**重新生成**」。

### 第 2 步：在浏览器扩展里粘贴 token

1. 点击浏览器右上角的**蓝色方块**图标（全视维 Web Clipper），会弹出一个小窗口。
2. 点窗口下方的「**⚙️ Settings**」展开设置。
3. 「**接收端点**」一栏保持默认 `http://127.0.0.1:33333/clip`（**不要改！** 除非你知道自己在干嘛）。
4. 「**X-Clip-Token**」一栏：右键 → 粘贴，把刚才复制的 token 粘进去。
5. 点「**保存**」。看到下方出现绿色「已保存」即成功。

### 第 3 步：测试连通

还在那个弹窗里，点上方的「**测试连接**」按钮：

- ✅ 显示绿色「**连接正常（HTTP 200）**」→ 大功告成，可以剪藏了！
- ❌ 显示红色失败 → 请看本手册「**七、常见错误对照表**」。

## 四、开始剪藏（4 种用法）

### 用法 ① 整页剪藏（最常用）

> 适合：完整保存一篇文章。

1. 在浏览器打开任意网页（比如一篇知乎文章、博客）。
2. 点工具栏的**蓝色方块**图标。
3. 点弹窗里的「**剪藏整页**」。
4. 弹窗显示绿色「**剪藏成功**」，桌面端右下角会冒出一条系统通知"剪藏完成 - 文章标题"。
5. 回到桌面端，会出现一个**和文章同名**的新页面，内容是 Markdown 格式的正文。

### 用法 ② 剪藏选区（只要片段）

> 适合：只想保存一段话/一个段落，不要整页。

1. 在网页上**用鼠标选中**想保存的文字。
2. 点工具栏的**蓝色方块**图标 → 点「**剪藏选区**」。
3. 这段文字会以引用块形式追加到**今日 journal** 里，并自动带上原文 URL。

### 用法 ③ 右键菜单剪藏（最快）

> 适合：碰到链接 / 选区 / 整页，懒得开扩展弹窗。

1. 在网页上**右键**。
2. 弹出的菜单里能看到三个全视维选项：
   - **剪藏整页到全视维**
   - **剪藏选区到全视维**（只在你选中文本时显示）
   - **剪藏此链接到全视维**（只在你右键一个链接时显示）
3. 点对应的菜单项即可，桌面端会立刻收到通知。

### 用法 ④ 在桌面端手动触发

> 适合：网页内容是从别处复制来的，手头没有 URL。

桌面端的命令面板（默认按 `Ctrl + Shift + P`）输入：

- `Clipper：快速剪藏到今日` —— 弹出 4 步表单（标题/URL/正文/标签），全填完就保存。
- `Clipper：剪藏选区 / 高亮` —— 适合手动整理一段引用。
- `Clipper：查看最近剪藏记录` —— 弹窗显示最近 15 条剪藏。

## 五、保存到哪里？格式是什么？

### 文件存放规则

| 情况 | 保存到 |
|------|-------|
| 有标题 + 整页模式 | **新建同名页面**（如果已存在则追加到末尾） |
| 无标题 或 选区模式 | **追加到今日 journal**（一个引用块） |
| 链接剪藏 | **追加到今日 journal**（一行 URL） |

### 保存的内容长啥样

举例：剪藏一篇文章后，新建页面会是这样：

```markdown
---
title: "深入理解 Rust 所有权"
source: https://example.com/rust-ownership
clipped: 2026-05-17 14:32
tags:
  - rust
  - 编程
---

# 深入理解 Rust 所有权

Rust 的所有权系统是一种独特的内存管理方式...

## 三条规则

- 每个值都有一个**所有者**
- 同一时刻只能有一个所有者
- 所有者离开作用域时，值被丢弃
```

最上面的 `---...---` 是 frontmatter，桌面端会自动识别标签、来源等。

## 六、平时怎么管理剪藏？

### 在桌面端查最近剪藏

打开「插件 → Clipper」面板，向下滚动到「**4. 最近请求**」：
- 这里能看到最近 50 条剪藏的时间、状态码、标题。
- HTTP `200` = 成功；`401` = token 错；`400` = 数据有问题。
- 点「清空」可以一键清空（**只清日志，不删笔记**）。

### 修改 token（怀疑泄露时）

1. 桌面端 Clipper 面板 → 点「**重新生成**」按钮，新 token 自动复制到剪贴板。
2. 浏览器右上角扩展图标 → Settings → 把新 token 粘进去 → 保存。
3. 旧 token 立刻失效，以后任何用旧 token 的请求都会被拒。

### 卸载扩展

- Chrome / Edge：进 `chrome://extensions` → 找到「全视维 Web Clipper」→ 点「移除」。
- 之后再删除解压出来的文件夹即可。

## 七、常见错误对照表

| 现象 | 原因 | 怎么办 |
|------|------|--------|
| 测试连接显示 `failed to fetch` 或 `连不上` | 桌面端没开，或被防火墙挡了 | ① 打开桌面端；② Windows 安全中心 → 防火墙 → 允许「全视维」走"私有网络" |
| 测试连接显示 `HTTP 401` | token 错了 | 回桌面端重新「复制」token，再到扩展 Settings 粘贴保存 |
| 剪藏成功但桌面端没反应 | 你没打开任何图谱 | 桌面端先打开/新建一个图谱再试 |
| 浏览器右上角找不到扩展图标 | 没固定到工具栏 | 点🧩拼图图标 → 找到扩展 → 点 📌 图钉固定 |
| 「剪藏选区」失败：当前页面没有选中文本 | 你忘了先选中 | 先用鼠标拖蓝选中文字，再点 |
| 重启浏览器后扩展不见了 | 火狐临时加载特性 | 重新走「火狐用户特别说明」，或换 Chrome / Edge |
| 桌面端显示 `请求体不是合法 JSON` | 扩展是旧版本 | 重新到 Release 页面下载最新 zip 覆盖 |
| 公司电脑装不了 / 提示企业策略禁止 | IT 关闭了开发者模式 | 找 IT 解锁；或换个人电脑 |

## 八、安全 & 隐私（一句话总结）

- 本扩展**只**和你电脑上的 `127.0.0.1:33333` 通信，**不连任何第三方服务器**。
- token 保存在浏览器本地（`chrome.storage.sync`），不会上传。
- 没有 token，外部任何人发请求都会被桌面端拒绝（401）。
- 桌面端 HTTP 服务**只监听本机**，局域网/同 WiFi 的其他设备**访问不到**。
- 全部源码公开在 GitHub `web-clipper-extension/` 目录，可随时审计。

## 九、进阶：手动用 curl 测试

如果你是技术人员，想验证管道：

```powershell
# 把 <你的token> 换成桌面端复制的字符串
curl -X POST http://127.0.0.1:33333/clip `
  -H "Content-Type: application/json" `
  -H "X-Clip-Token: <你的token>" `
  -d '{\"title\":\"测试\",\"url\":\"https://example.com\",\"body\":\"# Hello\\n\\n这是测试\",\"tags\":[\"test\"],\"mode\":\"page\"}'
```

返回 `{"ok":true,"page_name":"测试"}` 即成功。

健康检查（不需要 token）：

```powershell
curl http://127.0.0.1:33333/health
# → {"ok":true}
```
